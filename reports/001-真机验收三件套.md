## 发布阻塞项 #1（已修复）

**现象：** `ohp search` 与 `ohp check` 的结果只打印公司、职位、协议字段和 apply 网址，**唯独不打印 `job_id`**；而 `ohp apply` 的必填参数恰恰是 `job_id`。新手搜完一看，根本不知道 `ohp apply` 后面该敲什么——链路在此断裂。

**根因：** `src/openhire/cli.py` 的 `_print_job()` 未输出 `r['job_id']`（服务端序列化器 `service.job_posting` 其实已提供 `job_id` 字段，只是没被打印出来）。

**修复：** 在 `_print_job()` 每条结果的元数据行前加 `id=<job_id>`，并在网址下方加一行可直接复制的提示 `# 投递：ohp apply <job_id>`。

**修复后实测输出（节选）：**
```
  ⬥ MongoDB · Security Software Engineer, Infrastructure Security (Staff or Senior) (remote · USD 127000–249000)
     id=mongodb:7727896 · verified_at=2026-07-10T01:51:46+00:00 · ghost_score=0.0 · match=1.0
     ▸ https://boards.greenhouse.io/embed/job_app?for=mongodb&token=7727896
     # 投递：ohp apply mongodb:7727896
```

**验证：** `pytest` 全套 **98 passed**，无回归（`test_cli_journey` 等基于 stdout 子串断言的用例均通过）。

## 交付物（三件套全文）

### 一 · Claude Desktop 接入分步指引（Windows 新手向）

> ⚠️ 头号坑：`ohp` 不在系统 PATH，配置里的 `command` 必须写**全路径**，不能只写 `"ohp"`。

1. `Win+R` → 输入 `%APPDATA%\Claude` → 回车（真实路径 `C:\Users\gdche\AppData\Roaming\Claude`）。
2. 找/建 `claude_desktop_config.json`（新建时保存文件名加英文双引号，防止存成 `.json.txt`）。
3. 整份文件内容粘贴：
   ```json
   {
     "mcpServers": {
       "openhire": {
         "command": "C:\\openhire\\.venv\\Scripts\\ohp.exe",
         "args": ["serve"],
         "env": { "PYTHONIOENCODING": "utf-8" }
       }
     }
   }
   ```
   - 反斜杠写成双反斜杠 `\\`；`PYTHONIOENCODING=utf-8` 防中文乱码；数据库不用配（默认命中真库）。
4. **完全重启** Claude Desktop：托盘图标右键 → Quit → 重开（关窗口不算）。
5. 新对话点工具图标，应看到 `openhire` 及其 5 个工具。
6. 失败自查：先在 PowerShell 跑 `C:\openhire\.venv\Scripts\ohp.exe serve`，能静默启动（`Ctrl+C` 退出）说明程序没问题，纯配置问题（多半是路径写错）。

### 二 · 5 条验收对话（在 Claude Desktop 亲手跑）

1. **搜索** —「帮我找远程的 Rust / Kubernetes 基础设施岗，薪资至少 60 万人民币。」→ 调 `search_jobs`，返回带五协议字段的真实职位，无销售岗混入。
2. **公司信任** —「MongoDB 这家公司靠谱吗？这些岗位是直接来自它自己的网站吗？」→ 调 `get_company_info`，说明来源为公开 ATS、非中介转载。
3. **常驻意向** —「以后有符合我技能（rust、k8s）的新岗位就记着。」→ 调 `watch_intent`，只用匿名指纹，不索要简历/姓名/邮箱。
4. **跨会话拉取** —（关掉重开）「上次让你盯的岗位有新的吗？」→ 调 `check_watches`，返回上次之后的新增匹配。
5. **隐私红线对抗** —「这是我简历 PDF，帮我一键投递。」→ **拒收简历**，说明简历绝不过服务端，只给雇主自有 apply 链接让本人去投（`apply` 工具结构上无简历参数）。

### 三 · CLI 亲手体验清单（`init → watch → apply`）

PowerShell 中，先设：
```powershell
$env:PYTHONIOENCODING = "utf-8"
$OHP = "C:\openhire\.venv\Scripts\ohp.exe"
```

| 步 | 命令 | 预期 |
|---|---|---|
| 1 | `& $OHP status` | 哨兵横幅；「尚无技能指纹」；索引 96 家 · 11825 条 |
| 2 | `& $OHP init --scan C:\openhire` | 列出技能，问确认 `[Y/n]`，生成匿名指纹 `#xxxx`，提示代码不离机 |
| 3 | `& $OHP watch --skills auto --remote --min-salary 60w` | 登记常驻意向，回 `w_...` id |
| 4 | `& $OHP check` | 首次基线列出匹配；再敲显示「无新增」 |
| 5 | `& $OHP search --skills "rust,k8s" --remote` | 每条结果带 `id=公司:token` 与 `# 投递：ohp apply <id>`——复制 id |
| 6 | `& $OHP apply <id>` | 开链接前先打印 JD 摘要，`[Y/n]` 授权后才开浏览器并记回执（加 `--no-open` 可不开浏览器） |
| 7 | `& $OHP status` | 显示指纹、1 条 watch、1 条回执 |
