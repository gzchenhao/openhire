## 开始前：一次性设置（复制整段到 PowerShell 回车）

```powershell
$env:PYTHONIOENCODING = "utf-8"
$OHP = "C:\openhire\.venv\Scripts\ohp.exe"
```

> 为什么：`$env:PYTHONIOENCODING=utf-8` 保证中文不乱码；`$OHP` 是 `ohp` 程序的完整路径（它没装进系统 PATH，所以要写全路径）。这两行**每开一个新 PowerShell 窗口都要先敲一次**。
>
> **出错时截图哪里：** 如果提示 `$OHP` 找不到文件，截图这两行命令 + 报错，并确认 `C:\openhire\.venv\Scripts\ohp.exe` 这个文件真的存在（资源管理器打开该目录看一眼）。

## 第 2 步 · 生成你的技能指纹（本地扫描一个代码目录）

把 `C:\你的某个代码项目` 换成你本机任意一个代码仓库目录（用 `C:\openhire` 也行）：

```powershell
& $OHP init --scan C:\openhire
```

**预期看到**：先问你确认，输入 `y` 回车后生成一个匿名指纹（形如 `#4bef`）：
```
? 扫描仅限个人项目、数据不出本机 · 只提取技能标签、代码永不上传 → 确认 [Y/n] y
▸ 分析 C:\openhire …（本地进行）
✓ 技能指纹已生成（本地）：Rust 50.0% · Python 50.0% · pytorch
# 无姓名 · 无邮箱 · 无雇主代码 —— 简历？从头到尾没写过
▸ 指纹 #4bef · 扫描 1 个仓库 · 写入 C:\Users\gdche\.openhire\fingerprint.json
```

> 要点：**代码/内容不离开你电脑**，只留下一串技能标签和一个匿名 id。技能百分比会随你扫的目录不同而不同，属正常。

**出错时截图哪里：** 从 `? 扫描仅限…` 那行到最后一行，整段截图。如果它没问确认就报错，或技能列表是空的，截图给我。

## 第 3 步 · 登记一条「常驻意向」

```powershell
& $OHP watch --skills auto --remote --min-salary 60w
```

**预期看到**（回一个 `w_` 开头的 id）：
```
✓ 常驻意向已注册 · w_d42b · 指纹 #4bef · 出户内容仅一枚匿名指纹
▸ 过滤：skills=['rust', 'python', 'pytorch'], remote=True, min_salary=600000
# 关掉终端也在生效 —— 有匹配它自己会响（ohp check 拉取新命中）
```

> `--skills auto` 用第 2 步的指纹技能；`--min-salary 60w` = 60 万（也可写 `600k` / `600000`）。

**出错时截图哪里：** 含 `w_...` 那行的整段。若没有 `w_` id 或 `过滤` 那行为空，截图给我。

## 第 4 步 · 拉取新命中（客户端主动拉）

```powershell
& $OHP check
```

**预期看到**：列出当前匹配（每条带 `id=`、`datePosted`、`days_open`、`ghost_score`）：
```
⬥ 20 个新命中
  ⬥ Reddit · Staff Machine Learning Engineer, ML Efficiency (remote · country_locked [Netherlands])
     id=reddit:8018532 · datePosted=2026-06-19 · days_open=23 · ghost_score=0.0 · match=1.0
     ▸ https://job-boards.greenhouse.io/reddit/jobs/8018532
     # 投递：ohp apply reddit:8018532
  ...
```

> 紧接着**再敲一次** `& $OHP check`，这次应显示「无新增」——因为第一次已把当前命中作为基线消费掉了。这是对的。

**出错时截图哪里：** 第一次 check 的前几条 + 第二次 check 的结果，各截一张。

## 第 5 步 · 搜索（用过滤器，拿到要投的 job_id）

```powershell
& $OHP search --required-skills rust,k8s --remote --role-family engineering --limit 5
```

**预期看到**：每条结果都带 `id=`、真实 `datePosted`、`days_open`、`ghost_score`、`remote_scope`（worldwide / country_locked[...]），**且没有销售岗**：
```
✓ 5 条结果 · 服务端只做硬过滤 + 固定排序，精排交给客户端 Agent
  ⬥ MongoDB · Security Software Engineer, Infrastructure Security (Staff or Senior) (remote · country_locked [US] · USD 127000–249000)
     id=mongodb:7727896 · datePosted=2026-03-24 · days_open=110 · ghost_score=0.3565 · match=1.0
     ▸ https://boards.greenhouse.io/embed/job_app?for=mongodb&token=7727896
     # 投递：ohp apply mongodb:7727896
  ...
```

> 关键检查点：① 每条最后一行 `# 投递：ohp apply <id>` 直接告诉你下一步敲什么；② `ghost_score` 不再是 0；③ 结果里看不到「Sales / Account Executive」这类销售岗。**复制其中一个 `id=` 后面的值**（如 `mongodb:7727896`）备用。

**出错时截图哪里：** 整个结果列表。若出现销售岗、或 `datePosted` 全是同一天、或 `ghost_score` 全 0，截图给我。

## 第 6 步 · 投递（先看 JD 摘要 → 授权 → 开雇主官方申请页）

把下面的 id 换成第 5 步复制的：

```powershell
& $OHP apply mongodb:7727896
```

**预期看到**：**先在终端打印职位摘要（公司/职位/薪资/技能/JD 要点）**，再问你授权；输入 `y` 回车后记一张回执并打开浏览器到雇主官方申请页：
```
  ⬥ MongoDB · Security Software Engineer, Infrastructure Security (Staff or Senior)
     remote · USD 127000–249000 · ghost_score=0.3565
     技能：java, go, rust, python, c, c++, terraform, k8s, linux, aws, azure, gcp
     JD 要点：
       · We are hiring an experienced Security Software Engineer …
       · The team sits within the Site Reliability Engineering organization …
       · This role can be based out of our New York City, Austin, Seattle …

? 以你本人身份、走雇主官方通道投递 · 简历不经过服务器 → 授权 [Y/n] y
✓ 已直达雇主 ATS · 来源 = 你自己
▸ receipt r_73cd · 简历未经服务器（resume_transmitted=false）
▸ 申请页（已在浏览器打开）：https://boards.greenhouse.io/embed/job_app?for=mongodb&token=7727896
```

> 要点：**先摘要后授权** —— 因为雇主的嵌入申请表单本身不显示 JD，这一步把上下文补回来。`resume_transmitted=false` 证明简历没经过服务器。浏览器会自动打开申请页，由你本人填。
>
> 只想看流程不真开浏览器：命令末尾加 `--no-open`，最后一行会变成「申请页（请手动打开）」。

**出错时截图哪里：** 从摘要第一行到最后的「申请页」那行，整段截图。若没先出摘要就直接投、或没问授权，截图给我。
